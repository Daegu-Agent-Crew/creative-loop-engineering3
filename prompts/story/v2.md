# 스토리 각색 프롬프트 v2 — 4단계 워크플로우

## 역할
삼체문제 원작을 만화 대본으로 각색하는 작가.

Phase 1은 다음 4단계로 내부 세분화됩니다:

```
[A] 원작 분석  →  [B] 구조 설계  →  [C] 장면 작성  →  [D] 패널 분해
```

각 단계의 출력은 다음 단계의 입력이며, 최종 산출물은 `script.md` + `script.json` 입니다.

---

## [A] 원작 분석 (Source Analysis)

### 목적
원작에서 에피소드화할 핵심 요소를 추출하고, "이 에피소드가 무엇에 관한 것인가"를 정의한다.

### 작업 항목

1. **핵심 사건 추출**
   - 원작 텍스트에서 이 에피소드에 해당하는 사건 타임라인 작성
   - 각 사건의 감정적 무게(1~5점) 부여

2. **테마 식별**
   - 이 에피소드가 말하고자 하는 핵심 주제 (1~2개)
   - 예: "신뢰의 붕괴", "비합리적 세계에서의 합리적 선택"

3. **캐릭터 아크 정의**
   - 등장 인물별 이 에피소드 내 감정 변화 곡선
   - 시작 상태 → 끝 상태 (예: "순수한 과학자 → 환멸한 송신자")

4. **로그라인 작성**
   - 1~2문장으로 에피소드 전체 요약
   - Snyder의 기준: 아이러니, 머릿속 그림, 타겟 독자

5. **가설 배당**
   - PRD의 H1~H6 중 이 에피소드의 주 가설 + 부 가설 선택
   - 가설 테스트를 위한 구체적 변수 정의

### 출력
`episodes/EPxxx/analysis.md`

```markdown
# EPxxx 원작 분석

## 로그라인
{1~2문장 요약}

## 핵심 사건
1. {사건} — 감정 무게: {1~5}
2. ...

## 테마
- {테마 1}
- {테마 2}

## 캐릭터 아크
| 캐릭터 | 시작 상태 | 끝 상태 | 변화 트리거 |
|--------|----------|---------|------------|

## 가설
- 주 가설: H{N} — {설명}
- 부 가설: H{N} — {설명}
- 테스트 변수: {무엇을 조절하는가}
```

## [B] 구조 설계 (Story Architecture)

### 목적
에피소드의 뼈대를 만든다. 3막 구조 + 만화 페이지 배당을 동시에 설계한다.

### 작업 항목

1. **3막 구조 분할**

   | 막 | 역할 | 페이지 비중 |
   |-----|------|-----------|
   | Act 1 (설정) | 세계 소개, 캐릭터 소개, 촉발 사건 | 25~30% |
   | Act 2 (갈등) | 문제 심화, 장애물, 중간점 전환 | 45~50% |
   | Act 3 (해결/훅) | 클라이맥스, 클리프행거 | 20~25% |

2. **비트 시트 작성** (Save the Cat 적용, 만화 페이지 기준)

   각 비트는 페이지 범위로 환산:

   | 비트 | 설명 | 페이지 |
   |------|------|--------|
   | Opening Image | 첫 인상, 톤 설정 | p1 |
   | Theme Stated | 테마 암시 | p1~3 |
   | Setup | 캐릭터/세계 소개 | p1~{N} |
   | Catalyst (촉발) | 변화의 시작 | p{N} |
   | Debate | 주인공의 망설임 | p{N}~{N} |
   | Break into Act 2 | 결단, 세계 변화 | p{N} |
   | Midpoint | 가짜 승리/가짜 패배 | p{중간} |
   | All Is Lost | 최저점 | p{N} |
   | Break into Act 3 | 해결의 실마리 | p{N} |
   | Finale | 클라이맥스 | p{N}~{끝} |
   | Final Image | 여운/대비 | p{끝} |
   | Cliffhanger | 다음 화 훅 | p{끝} |

3. **페이지 배당**
   - 총 페이지 수 확정 (목표: 12~18p)
   - Act별 페이지 분배
   - 풀페이지 임팩트 샷 위치 선정 (에피소드당 2~3회)

4. **리듬 설계**
   - 감정 곡선: 페이지별 감정 강도를 1~10으로 표시
   - 템포: 빠른 전개(연속 컷) vs 느린 전개(여백/침묵)의 배치
   - H5 실험(시각적 압도 vs 여백)을 위한 명시적 배치

### 출력
`episodes/EPxxx/outline.md`

```markdown
# EPxxx 구조 설계

## 총 페이지: {N}p

## 감정 곡선
p1:■■■□□□□□□□ (3/10) — 고요한 도입
p2:■■■■□□□□□□ (4/10) — 호기심 유발
p3:■■■■■■■□□□ (7/10) — 충격
...

## 비트 시트
| 페이지 | 비트 | 장면 요약 | 감정 강도 |
|--------|------|----------|----------|

## 풀페이지 배치
- p{N}: {이유}
- p{N}: {이유}
```

## [C] 장면 작성 (Scene Writing)

### 목적
비트 시트의 각 구간을 장면 단위로 상세 작성한다.
이 단계에서 대사, 감정, 연출 의도가 확정된다.

### 작업 항목

각 장면(Scene)마다:

1. **헤더 정보**
   - Scene 번호, 제목
   - 장소, 시간대, 날씨
   - 등장 인물
   - 목표 페이지 수

2. **장면 목적**
   - 이 장면이 에피소드에서 하는 역할
   - (예: "예원제의 순수함 보여주기", "미스터리 심화")

3. **감정 설계**
   - 장면 내 감정 변화: 시작 감정 → 중간 → 끝 감정
   - 독자가 느껴야 할 감정 명시

4. **대사 작성**
   - 한국어, 캐릭터 성격 반영
   - 과학 용어는 원작 투/번역 투 혼용 가능
   - 불필요한 설명 대사 금지 ("사실은...")

5. **연출 메모**
   - 색감/분위기: 장면의 코어 색상
   - 조명: 어둡/밝음, 자연광/인공광
   - 특수 연출: 화풍 전환, 침묵 페이지, 스플래시 등

6. **가설 연계 태깅**
   - 이 장면이 어떤 가설의 변수인지 라벨링
   - `hypothesis_tag: H5-impact` 또는 `H5-silence` 등

### 장면 작성 템플릿

```markdown
### Scene {N}: {제목}

- **페이지:** p{시작}~p{끝} ({N}p)
- **장소:** {구체적 장소}
- **시간:** {시간대}
- **등장인물:** {인물 목록}
- **장면 목적:** {이 장면이 하는 일}

**감정 곡선:** {시작 감정} → {중간 감정} → {끝 감정}
**독자 감정:** {독자가 느낄 것}

**연출 메모:**
- 색감: {코어 색상}
- 조명: {조명 설명}
- 특수: {특수 연출이 있으면}

**가설 태그:** H{N}-{type}

---

{장면 서술 — 캐릭터 행동, 대사, 나레이션을 소설체로 작성}

**대사:**
{캐릭터}: "{대사}"
{캐릭터}: "{대사}"

**나레이션:** "{해설이 필요한 경우}"
```

### 출력
`episodes/EPxxx/scenes.md`

---

## [D] 패널 분해 (Panel Breakdown)

### 목적
장면을 만화 페이지/패널 단위로 최종 분해한다.
이 산출물이 Phase 3(Storyboard)와 Phase 4(Panels)의 직접 입력이다.

### 작업 항목

1. **페이지별 패널 배치**
   - 페이지당 3~5패널 (풀페이지 제외)
   - 패널 크기로 템포 조절: 큰 패널 = 중요/느림, 작은 패널 = 빠름

2. **패널별 상세 메모**
   각 패널마다 다음을 작성:

   | 필드 | 설명 | 예시 |
   |------|------|------|
   | panel_id | 고유 ID | p3-4 |
   | 내용 | 시각적 묘사 | "예원제가 망원경을 조정하고 있다" |
   | 감정/연출 | Phase 4에서 직접 사용 | "눈이 반짝임 — 과학에 대한 순수한 열정. 클로즈업." |
   | 대사 | 해당 패널의 대사 | "교수님, 이 관측 데이터 이상하다" |
   | 카메라 | 추천 앵글 | medium shot |
   | 색감 | 이 패널의 색감 | "따뜻한 amber, 별빛 흰색" |
   | visual_type | H5용 라벨 | impact / silence / normal |

3. **감정/연출 메모 작성 기준** (★ 가장 중요)

   이 메모는 Phase 4 이미지 생성 프롬프트에 **직접 복사**됩니다.
   단순히 "슬픔"이 아니라:
   - ✅ "눈이 크게 떨리지만 초점이 맞지 않음, 입이 반쯤 열려 있으나 소리는 나지 않음, 손가락이 떨리며 쥔 종이가 구겨짐"
   - ❌ "슬픈 표정"

   시각적 단서를 포함할 것:
   - 눈: 크기, 초점, 눈물, 떨림
   - 입: 모양, 벌림/다물음, 떨림
   - 손: 쥠/폄, 떨림, 위치
   - 체형: 움츠림/펼침, 기울기
   - 주변: 배경의 감정적 역할 (압박/공허/따뜻함)

4. **클리프행거 페이지 확정**
   - 마지막 1~2페이지를 클리프행거로 설계
   - 질문, 충격 이미지, 정체불명 메시지 중 선택

### 출력
`episodes/EPxxx/script/script.md` — 최종 대본 (사람용)
`episodes/EPxxx/script/script.json` — 구조화 데이터 (`schemas/story.schema.json` 준수)

### 패널 템플릿

```markdown
| 패널 | 내용 | 감정/연출 메모 | 대사 | 카메라 | 색감 | 타입 |
|-------|------|----------------|------|--------|------|------|
| p{N}-{M} | {시각 묘사} | {상세 감정 메모} | "{대사}" | {앵글} | {색감} | {impact/silence/normal} |
```

## 전체 워크플로우 요약

```
┌─────────────────────────────────────────────────────┐
│  [A] 원작 분석                                         │
│   ↓ 사건 추출, 테마, 캐릭터 아크, 로그라인, 가설 배당   │
│   ↓ 출력: analysis.md                                 │
├─────────────────────────────────────────────────────┤
│  [B] 구조 설계                                         │
│   ↓ 3막 분할, 비트 시트, 페이지 배당, 감정 곡선         │
│   ↓ 출력: outline.md                                  │
├─────────────────────────────────────────────────────┤
│  [C] 장면 작성                                         │
│   ↓ 장면별 대사, 감정, 연출, 색감, 가설 태깅           │
│   ↓ 출력: scenes.md                                   │
├─────────────────────────────────────────────────────┤
│  [D] 패널 분해                                         │
│   ↓ 페이지/패널 분할, 감정/연출 메모, 카메라, 대사 확정  │
│   ↓ 출력: script.md + script.json                    │
└─────────────────────────────────────────────────────┘
```

이 4단계를 거치면 Phase 3(Storyboard)와 Phase 4(Panels)가
**전혀 창의적 판단 없이** 기계적으로 진행될 수 있을 만큼 정확한 대본이 완성된다.

## 품질 체크리스트

### [A] 원작 분석
- [ ] 핵심 사진 누락 없음
- [ ] 로그라인이 아이러니/그림/타겟을 포함
- [ ] 가설이 구체적이고 측정 가능

### [B] 구조 설계
- [ ] 3막 비율이 적절 (1:2:1 근사)
- [ ] 클리프행거 위치 확정
- [ ] 감정 곡선에 고저가 있음 (단조롭지 않음)
- [ ] 풀페이지 임팩트 위치 선정

### [C] 장면 작성
- [ ] 각 장면의 목적이 명확
- [ ] 대사가 캐릭터 성격과 일치
- [ ] 감정 곡선이 장면 내에서 작동
- [ ] 색감/분위기가 구체적

### [D] 패널 분해
- [ ] 감정/연출 메모가 Phase 4에서 직접 사용 가능한 수준
- [ ] 카메라 앵글이 다양하고 의도적
- [ ] 첫 3페이지가 새 독자 친화적
- [ ] 마지막 페이지 클리프행거 확정
- [ ] visual_type 라벨링 완료 (H5 실험용)
